\chapter{Conclusions}
\label{chap:conclusions}

This thesis set out to close part of the marketing-quality gap between large brands and the independent retailers that compete with them, by building a production-grade cloud platform that turns a shop's product photographs, brand identity and a stream of real-world context signals into ready-to-publish marketing assets. The platform was delivered as a cross-platform React Native client backed by an ASP.NET Core service on Microsoft Azure, with three technical contributions sitting on top of the architectural baseline: a multi-stage daily recommendation engine grounded in retrieved context and a fine-tuned writer, a hybrid generative--deterministic catalogue pipeline that preserves product fidelity pixel for pixel, and a tiled super-resolution path that turns model-resolution assets into print-ready PDFs.

The thread running through the contributions is that the gap between an off-the-shelf model and an output a small retailer is willing to publish is rarely closed by a bigger model. It is closed by the protocol that sits around the model: a typed prompt chain in the recommendation case, an outline-as-contract in the catalogue case, and fixed paper sizes with a hard $300\,$dpi target in the print case. In each instance the deterministic machinery around the AI call is what makes the AI call's output reliable enough to ship.

\section{Future improvements}
\label{sec:future-work}

Several extensions would meaningfully strengthen the platform.

\begin{itemize}
    \item Self-serve billing via Stripe: replace the admin-grant flow with Stripe Checkout for one-off credit packs and Stripe Billing for subscriptions, reconciling charges against the existing ledger through webhooks.
    \item Engagement feedback loop: pull per-post reach, engagement and click-through from the connected social channels and feed them back into the recommendation engine as a per-shop signal, so future ideas favour patterns that have worked for that retailer.
    \item Per-shop writer adaptation: use periodic LoRA refreshes on each shop's accepted-versus-rejected ideas to let the writer converge to the shop's voice without retraining the base model; the hot-swappable adapter infrastructure is already in place.
    \item Short-form video generation: extend the pipeline beyond stills to $5$ to $15$\,s vertical clips for Reels, TikTok and Shorts; the hybrid contract translates naturally, with the compositor placing real product photos onto each generated frame.
    \item Scheduled and automated publishing: let the retailer mark an accepted idea for auto-publication at a chosen time, with retry on platform failure, turning the platform from an authoring tool into a daily marketing operation that runs without operator help.
    \item Print-on-demand fulfilment: partner with a print-on-demand provider so the A4 export becomes an in-app order for shipped flyers, which is the form most independent retailers actually need.
    \item Controlled benchmark for the recommendation engine: run a rigorous study with a power-sized frozen evaluation set, blind human raters from the target audience, and a per-failure-mode breakdown, in place of the current LLM-as-judge and small pairwise pass.
\end{itemize}
